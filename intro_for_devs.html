Introduction to Blue (If You Already Know How to Code)

Blue is a programming language that is 100% in the browser and compiles into JavaScript. It runs very similarly to JavaScript, except has different syntax. The main advantages of Blue are how it handles time (same as JavaScript), and how it makes graphics very easily. It is mainly used for basic game development when learning how to code, and for other elementary projects. After completely understand Blue, it is easy to move to JavaScript.

How Does Blue Work?

Blue is split into two parts: the compiling and the library. Blue was written entirely in JavaScript.

Compiling:
Once you click the RUN button, Blue has to turn your code into JavaScript. So it just switches all of the syntax and turns it into JavaScript. To see how Blue compiles the code, see this JavaScript file. It is pretty short, just over 100 lines. The compiler isn't always perfect because it wasn't created the same way that most compilers are made, but it still works well.

The Library:
Once your code has been converted into JavaScript, it can't just run as plain JavaScript because Blue has a lot of added features. Aside from the compiler, Blue is just a JavaScript library, meaning that it just adds extra features to JavaScript. Blue uses the JavaScript and HTML canvas to create the graphics. You can view this part of the code at this JavaScript file.

The Text Editor
The rest of the code for Blue is for saving projects and making the text editor. Blue saves your projects in the JavaScript localStorage. Because of this, you never have to sign in and your projects are just saved locally (if your write code on one computer, it will not be accessible on any other computer). The text editor and syntax highlighter was created entirely from scratch by me. The syntax highlighter uses an invisible textarea with a div directly on top of it (the div is the part with the colorful code).
